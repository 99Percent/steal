Helpers that make exporting to CJS projects easier.

"+cjs": { ... OVERWRITES ... }

Adds helpers that export modules to a CJS format.

Use

Adding "+cjs" to a ExportOutput name will mixin default ExportOutput and TransformOptions values that export modules to a CommonJS format.

outputs: {
  "+cjs": {},
  "minified+cjs": {
    minify: true
  }
}

This mixes in the following default values:

  • graphs - Writes out System.main and all of its dependencies.
  • format - Writes out all modules as CJS.
  • normalize - Leaves moduleName references that reference modules in node_modules alone; makes all other moduleName references relative.
  • dest - Writes out each module in [baseURL]/dist/cjs.
  • ignore - Ignores everything in node_modules.

You can overwrite or alter the behavior of these default values by adding a value in the ExportOutput. The following writes out dest in a different location:

outputs: {
  "+cjs": {
       dest: __dirname+"/cjs"
  }
}

The behavior for overwriting ExportOutput values is documented in the default value API pages.
